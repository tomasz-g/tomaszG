2048 Game
Principles of Computing in Python which is 2 of 3 in the Fundamentals of Computing Specialization course.
by Joe Warren, Scott Rixner, John Greiner, Stephen Wong -- Rice Univercity
project by TG
2048 is a simple grid-based numbers game. The object of the game is to combine tiles with the same number to make larger numbered tiles. You "win" when you create a 2048 tile. You will have a grid of 16 tiles (which can be modify in this implementation) and two numbers on start: '2' 90% of the time and '4' 10% of the time. On each turn, you may slide all of the tiles on the board in one direction (left, right, up, or down).When you do so, all of the tiles on the board slide as far as they can go in the given direction leaving no empty squares between the tiles. Further, if two tiles of the same number end up next to each other, they merge to form a new tile with twice the value. If any tile slide, new random tile will be given with the same manner as on start.If no tiles would combine or slide in a given direction, then that is not a legal move, and you cannot make that move on the current turn. If there are no free tiles on the grid, the game ends.

2048 - HOW TO PLAY?

- use arrows to slide tiles
- hit 'New Game' button to reset Game
- hit 'How to Play' button to see Game instruction
- hit 'Back to Game' button to hide Game instruction
- change grid size
change value of HEIGHT and/or WIDTH variable in the code
'2' <= HEIGHT, WIDTH <= '10' # not tested above '10'

* Graphical user interface "GUI class" provided by instructors and modified by TG (add game instruction)

Word Wrangler

Loyd's Fifteen puzzle - solver and visualizer
is a sliding puzzle that consists of a frame of numbered square tiles in random order with one tile numberded 0. The object of the puzzle is to place the tiles in order by making sliding moves that use the empty space (tile numberded 0).

HOW TO PLAY?
Note that solved configuration has the blank (zero) tile in upper left
Use the arrows key to swap this tile with its neighbors.
Some configurations (if use random shuffle) are not solvable by this program, if get message "2x2 puzzels can't be solved" on the console, some or all top-left 4 tiles will remain unsorted.

Zombie Apocalypse

Simulation of zombies (red sqares) and humans (green squares) interacting on a grid. Zombies chase humans and humans flee from zombies which can only move up, down, left or right in one step of the simulation and humans are more agile and can move in these four directions as well as the four neighboring diagonal directions. If a zombie catches a human by positioning itself in the same cell, human become zombie (change for purple for a moment and then red permanently). Some of the cells (black squares) in this grid will be marked as impassable and restrict zombie/human movement so that they can not move through these cells.

How To Play

- popup windows must be enabled!!
- Hit PLAY icon to Start (left top corner)
- mouse click on canvas to add obstacle, zombie or human
("Mouse click:" button to change add option)
- Clear all button to reset the game and clear the city
- Humans flee button to move humans one step of simulation
- Zombies stalk button to move zombies one step of simulation
- Dark City button to add random obstacles, zombies and humans
- Start Game to start auto simulation
- you can add zombies, humans, obstacle or random any time during the simulation
